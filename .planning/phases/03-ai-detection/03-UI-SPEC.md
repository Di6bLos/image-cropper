## Component Inventory (Phase 3 Additions)

### Auto Crop Button

| Property | Value |
|----------|-------|
| Type | IconButton.vue variant |
| Icon | `sparkles` or `magic` (inline SVG — new) |
| Size | lg (44px) |
| Placement | Crop workspace toolbar, next to existing ratio controls |
| Label | "Auto crop" |
| States | default, hover, active, disabled (during processing), loading |

**Behavior:**
- Click triggers Smartcrop.js subject detection on selected image
- While processing: button disabled, shows spinner icon, label changes to "Detecting subject..."
- On success: crop window repositions to detected focal point
- On failure: falls back to center-crop, shows toast "Center crop applied"

### AI Processing Indicator

| Property | Value |
|----------|-------|
| Type | Inline spinner (CSS animation) |
| Position | Inside Auto crop button, replacing icon during processing |
| Animation | 360-degree rotation, 800ms ease-in-out, infinite |
| Color | `var(--accent)` |

### Toast Notification (Fallback)

| Property | Value |
|----------|-------|
| Type | Non-blocking toast |
| Position | Bottom-center of crop workspace |
| Duration | 3000ms auto-dismiss |
| Content | "Center crop applied" or "Detection unavailable" |
| Style | Semi-transparent dark background, white text, rounded corners (6px) |

## Interactions

### Auto Crop Flow

```
User clicks "Auto crop"
    │
    ├─► Button enters loading state (disabled + spinner)
    │
    ├─► Smartcrop.js loaded from CDN (if not already cached)
    │
    ├─► AI detection runs in Web Worker
    │
    ├─► [Success] Crop window repositions to detected focal point
    │        └─► Toast: "Center crop applied" (if fallback was triggered)
    │
    └─► [Failure] Center-crop applied
             └─► Toast: "Detection unavailable"
```
