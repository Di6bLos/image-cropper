## Key Implementation Patterns (from RESEARCH.md)

### SSR-Safe Composable Guard (D-02)
```typescript
export function useSomething() {
  if (!process.client) return // Early return for SSR
  // Browser-only code here
  onMounted(() => { ... })
}
```

### useBlobRegistry Pattern (D-05)
- Track all URL.createObjectURL calls in a Set
- Provide create(), revoke(), revokeAll() methods
- Auto-cleanup on component unmount via onUnmounted

### useCanvasPool Pattern (D-06)
- Pre-allocate pool of canvases at init
- Default pool size: Math.max(2, navigator.hardwareConcurrency ?? 2)
- Use OffscreenCanvas if available, fallback to regular canvas
- Provide acquire(), release(), dispose() methods
- Auto-dispose on component unmount

### useImageWorker Pattern (D-04)
- Manage single Worker instance lifecycle
- Expose isProcessing as readonly Ref
- postMessage with optional Transferable array
- Terminate on unmount

### Worker Message Protocol
- Uses postMessage with ImageBitmap transferables
- bitmap.close() must be called after drawImage() to release GPU memory
