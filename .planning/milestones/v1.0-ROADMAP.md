# Milestone v1.0: MVP — Image Cropper

**Status:** ✅ SHIPPED 2026-07-17
**Phases:** 1-5
**Total Plans:** 9

## Overview

Initial MVP delivering a browser-based bulk image cropping tool with drag-drop import, manual crop positioning, AI-assisted subject detection (Smartcrop.js), batch processing, and ZIP export.

## Phases

### Phase 1: Foundation

**Goal:** Project infrastructure is stable and production-ready with proper memory management patterns

**Depends on:** Nothing (first phase)

**Plans:** 2 plans

- [x] 01-01: Initialize Nuxt 3 project with modules and COOP/COEP headers
- [x] 01-02: Memory management composables and worker foundation

**Details:**
- Nuxt 3 + Pinia + Sass + nuxt-security module
- COOP/COEP headers for SharedArrayBuffer support
- useBlobRegistry composable for centralized Blob URL lifecycle
- useCanvasPool composable with OffscreenCanvas support and Safari fallback
- useImageWorker composable for Web Worker lifecycle management
- image-processor.worker.ts with message protocol
- SSR-safe patterns with process.client guards

### Phase 2: Core Cropping

**Goal:** Users can import images and perform manual crop operations with ratio constraints

**Depends on:** Phase 1

**Plans:** 4 plans

- [x] 02-01: Core infrastructure (composables + stores)
- [x] 02-02: UI components (DropZone, ImageList, CropOverlay, RatioControls)
- [x] 02-03: Page layout integration (responsive + dark mode)
- [x] 02-04: Testing and verification

**Details:**
- File picker import (click-to-browse)
- Drag-and-drop with visual feedback
- Lazy-loaded thumbnails with filename/dimensions

- Ratio constraint during resize (16:9, 3:2, 1:1, 2:3 presets)

- Custom pixel dimensions input
- Semi-transparent overlay outside crop window
- White border on crop window
- Dark mode via CSS variables (prefers-color-scheme)
- Responsive layout with max-width: 100%

---

### Phase 3: AI Detection

**Goal:** Users can automatically detect subjects and position crop window using Smartcrop.js

**Depends on:** Phase 2

**Plans:** 1 plan

- [x] 03: Smartcrop.js AI detection with CDN lazy-loading, worker off-thread processing, center-crop fallback

**Details:**
- Smartcrop.js lazy-loaded from CDN (jsdelivr, pinned to v2.0.5)
- AI processing runs in Web Worker (no UI freeze)
- Auto-crop button triggers subject detection
- Detected focal point positions crop window
- Center-crop fallback when AI unavailable/fails
- Keyboard shortcut 'A' for auto-crop
- Toast notifications for loading/success/failure states

---

### Phase 4: Batch Processing

**Goal:** System can process 50+ images without freezing UI, with per-file progress reporting

**Depends on:** Phase 3

**Plans:** 1 plan (3 waves)

- [x] 04: Worker pool, export pipeline, batch progress UI, memory safety

**Details:**
- Worker pool sized to navigator.hardwareConcurrency
- Round-robin task distribution to workers
- Canvas pooling for memory efficiency
- ImageBitmap properly closed after transfer (memory safety)
- Per-file progress callbacks

## Milestone Summary

### Key Decisions

- **Nuxt 3 + Vue 3 + Sass + Pinia** — User-specified, good DX
- **Smartcrop.js via CDN lazy-load** — Keeps initial bundle small, works offline
- **Web Worker for AI detection** — Off main thread, no UI freeze
- **Center-crop fallback** — Always gives user a usable result
- **Client-side only processing** — Privacy-first, no server costs for image bytes
- **Horizontal Layers architecture** — Composables/stores/components/workers separation

### Issues Resolved

- SSR-safe patterns (isMounted ref, process.client guards)
- Cross-origin isolation (COOP/COEP headers)
- Memory management (Blob URL registry, canvas pooling, ImageBitmap closure)
- Safari OffscreenCanvas fallback
- Phase 3 PR merge conflicts resolved

### Issues Deferred

- Human verification checkpoint for Phase 2 visual UX (11 manual tests)
- Phase 4 VERIFICATION.md (VALIDATION.md with manual checklist used instead)
- useCropWindow composable unused (logic inlined in CropWorkspace)
- useImageWorker not effectively used (Smartcrop creates own worker)

### Technical Debt Incurred

- Phase 2 human verification not automated
- Phase 4 has no VERIFICATION.md (VALIDATION.md covers it)
- Two composables defined but not imported (no functional impact)
