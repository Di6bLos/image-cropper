# Requirements Archive: Milestone v1.0

**Archived:** 2026-07-17
**Status:** All v1 requirements implemented

## v1 Requirements — All Implemented

### Import

- [x] **IMPT-01**: User can import images via file picker (click to browse) — ✅ DropZone file input + Browse button
- [x] **IMPT-02**: User can import images via drag-and-drop onto a visible drop zone — ✅ DropZone drag handlers + isDragging state
- [x] **IMPT-03**: Drop zone provides visual feedback on drag-over (highlight/border change) — ✅ .dropzone--dragging class changes border/background
- [x] **IMPT-04**: Imported images generate lazy-loaded thumbnails for the preview list — ✅ generateThumbnail via createImageBitmap + canvas pool

### Preview

- [x] **PRVW-01**: All imported images appear in a scrollable bulk preview list — ✅ ImageList v-for over imageStore.images
- [x] **PRVW-02**: Each list item shows thumbnail, filename, and original dimensions — ✅ ImageListItem renders thumbnail/filename/dimensions
- [x] **PRVW-03**: User can click an image in the list to select it for cropping — ✅ select event → imageStore.selectImage

### Crop Window

- [x] **CROP-01**: Selected image displays a draggable crop window overlay — ✅ CropOverlay renders when selectedImage exists
- [x] **CROP-02**: Crop window has visible handles at corners and edges for resizing — ✅ 4 CropHandle components at nw/ne/sw/se
- [x] **CROP-03**: Crop window constrains to selected aspect ratio while resizing from corners — ✅ onResizeMove applies ratio constraint
- [x] **CROP-04**: Area outside the crop window is dimmed with a semi-transparent overlay — ✅ clip-path polygon with rgba(0,0,0,0.65)
- [x] **CROP-05**: Crop window has a visible white border to distinguish it from the image — ✅ box-shadow: 0 0 0 2px white
- [x] **CROP-06**: Crop window position is clamped within image bounds — ✅ Bounds clamping in onResizeMove/onDragMove

### Ratio & Size

- [x] **RATIO-01**: User can select from aspect ratio presets: 16:9, 3:2, 1:1, 2:3 — ✅ 4 preset buttons
- [x] **RATIO-02**: Selecting a preset immediately constrains the crop window to that ratio — ✅ watch effectiveRatio → applyRatioConstraint
- [x] **RATIO-03**: User can enter a custom ratio as "W:H" (e.g., "7:4") and apply it — ✅ Custom section with W and H inputs
- [x] **RATIO-04**: User can enter custom pixel dimensions (e.g., "1200x800") for export size — ✅ Pixel mode with Width x Height inputs
- [x] **RATIO-05**: Custom ratio and pixel size inputs are mutually exclusive modes (radio/toggle) — ✅ Tab interface with mode: 'ratio' | 'pixel'

### AI Detection

- [x] **AIDT-01**: "Auto crop" button runs AI subject detection on the selected image — ✅ AutoCropButton triggers smartcrop.detect
- [x] **AIDT-02**: Smartcrop.js is lazy-loaded on first use from CDN (not bundled) — ✅ CDN lazy-load via importScripts in worker
- [x] **AIDT-03**: Detected focal point positions the crop window automatically — ✅ Detected crop applied via cropWindow.setCrop
- [x] **AIDT-04**: If AI detection fails or is unavailable, center-crop is used as fallback — ✅ calculateCenterCrop() fallback
- [x] **AIDT-05**: AI processing runs off the main thread (Web Worker) — ✅ aiDetect in Web Worker off main thread

### Export

- [x] **EXPT-01**: "Export" button opens a format/quality options modal before downloading — ✅ CropWorkspace onExport → useExportModal().show()
- [x] **EXPT-02**: Modal offers format selection: JPG, PNG, WebP — ✅ ExportModal format radio (JPG/PNG/WebP)
- [x] **EXPT-03**: Modal offers a quality slider (1-100) for lossy formats (JPG, WebP) — ✅ Quality slider v-if="options.format !== 'png'"
- [x] **EXPT-04**: Export applies the current crop settings to all images in the list — ✅ cropX/Y/Width/Height passed to worker
- [x] **EXPT-05**: Exported files preserve original filenames with an optional "_cropped" suffix — ✅ buildExportFilename with suffix
- [x] **EXPT-06**: Multiple images are bundled into a single ZIP download — ✅ generateZip bundles all results
- [x] **EXPT-07**: ZIP generation uses streaming to avoid memory exhaustion on large batches — ✅ streamFiles: true in generateAsync

### UI/UX

- [x] **UIUX-01**: Dark mode support using CSS variables, respecting system preference — ✅ _variables.scss prefers-color-scheme: dark
- [x] **UIUX-02**: Canvas and crop window scale responsively to the browser window size — ✅ max-width: 100%, responsive grid
